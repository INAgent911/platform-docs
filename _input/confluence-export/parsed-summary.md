Database Postgres Schema designed as a YAML Schema is automatically checked and updated from YAML on login Customized schema is also defined as YAML SCHEMA must be up to date Once tables and fields are defined, it's really bad to remove them. Deprecation strategies must allow for 1 year, and it can be undone. Text fields can be vectorized Blobs should not be in the database. they are links to Blob Storage through hashkeys. If an image is used twice, it's one blob. Permissions are record level to filter what is available to AI It's multitenant capable so MSP's unique id's are in the table. If a table is called customers, make a customers_custom table, defined by YAML.
